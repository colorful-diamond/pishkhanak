کد پیگیری، رشته ای اختیاری با طول حداکثر ۴۰ کاراکتر شامل حرف و عدد. در صورت ارسال trackId، فراخوانی سرویس خود را با همین مقدار استعلام و پیگیری کنید.(در گزارش فراخوانی سرویس ها با همین رشته نتیجه را ببینید). در صورتیکه که این فیلد را ارسال نکنید یک رشته UUID برای این فراخوانی در نظر گرفته میشود و در پاسخ فراخوانی برگردانده میشود. * example: ```getUserinfo781432``` * **inquiryTrackId:** اختیاری (string) &#x202b; کد پیگیری سرویس پین شارژ فراخوانی شده * example: ```heyhey``` ####Results Format ##### Successful result format (status code 200) ```````````````` { "responseCode": "FN-PGFH-20000100000", "trackId": "8267738a-9efd-4995-8581-63b638a40f66", "result": { "status": "SUCCESS", "pin": "111222333444555666777", "serial": "29194000358477769" }, "status": "DONE" } ```````````````` * **result:** آبجکتی از پاسخ سرویس شامل: * **status:** وضعیت خرید شارژ؛ دارای دو مقدار SUCCESS و FAILED * **pin:** مقدار پین در صورت SUCCESS بودن status * **serial:** مقدار سریال در صورت SUCCESS بودن status #### UnSuccessful result format در اینجا میتوانید لیست خطاهای سرویس ها را مشاهده کنید [لیست خطاها](/doc/generic-errors.html)
