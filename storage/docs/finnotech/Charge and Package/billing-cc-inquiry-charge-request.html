###GET /billing/v2/clients/{clientId}/chargeInquiry * **شرح: ** وب سرویس شارژ مستقیم این امکان را به مشترکان می دهد تا بدون نیاز به وارد کردن کد رمز و بطور مستقیم خط خود را شارژ نمایند. اپراتور های تحت پوشش شامل: همراه اول می باشد. . * **اسکوپ: ** ```billing:charge-inquiry:get``` * **رویکرد: ** ```Client_Credential``` * **<a href="ResponsecodesBilling.pdf"> دانلود مستندات ریسپانس کد ها</a> ** ```````````````` {address}/billing/v2/clients/{clientId}/chargeInquiry?trackId={trackId}&orderId={orderId} ```````````````` <div class="sandboxaddress sandbox" id="sandbox" style="float:right" style="display: none" > https://sandboxapi.finnotech.ir<b><span class="finnotech-sandbox-address-text"> :address </span></b> </div> <div class="production-address production" style="float:right" style="display: block"> https://api.finnotech.ir <b> :address </b> </div> #### Headers مقادیر زیر باید در هدر قرار بگیرد ```````````````` Authorization : Bearer {Token} ```````````````` * برای فراخوانی این سرویس لازم است پس از دریافت توکن Client Credential سرویس را فراخوانی نمایید. #### URI Parameters * **clientId:**شناسه کلاینت #### Query Parameters * **trackId:** اختیاری (string) &#x202b; کد پیگیری، رشته ای اختیاری با طول حداکثر ۴۰ کاراکتر شامل حرف و عدد. در صورت ارسال trackId، فراخوانی سرویس خود را با همین مقدار استعلام و پیگیری کنید.(در گزارش فراخوانی سرویس ها با همین رشته نتیجه را ببینید). در صورتیکه که این فیلد را ارسال نکنید یک رشته UUID برای این فراخوانی در نظر گرفته میشود و در پاسخ فراخوانی برگردانده میشود. * example: ```getUserinfo876543``` * **orderId:** اجباری (string) &#x202b; شماره پیگیری درخواست شارژی که باید پیگیری شود * example: ```101``` ####Results Format ##### Successful result format (status code 200) ```````````````` { "responseCode": "FN-BGTP-20001300000", "trackId": "e325b243-9d5b-4fab-974e-123191567660", "result": { "mobile": "", "completed": true, "status": "Ok", "finalStatus": "Confirmed" }, "status": "DONE" } ```````````````` می توانید کلید خروجی های سرویس را به تفکیک نوع سرویس از [اینجا](./asset/InfoKey.xlsx) دریافت کنید * **result:** آبجکتی از پاسخ سرویس شامل: * **completed:** وضعیت درخواست * **status:** وضعیت درخواست * **message:** پیام مربوط به درخواست * **finalStatus:** وضعیت درخواست پیگیری شده * **Requested:** درخواست داده شده * **Sent:** ارسال شده * **SendingError:** خطا در ارسال * **Confirm:** درخواست تایید شده * **Confirmed:** تراکنش انجام شده * **ConfirmingError:** خطا در انجام تراکنش * **status: ** وضعیت فراخوانی سرویس * DONE: فراخوانی موفق سرویس * FAILED: فراخوانی ناموفق سرویس * **trackId: ** کد پیگیری، اگر ارسال شده باشد همان مقدار و در غیر اینصورت یک رشته تصادفی تولید و برگردانده میشود * **responseCode: ** کد پاسخ فینوتک * **error:** جزییات خطا (در صورت بروز خطا) #### UnSuccessful result format در اینجا میتوانید لیست خطاهای سرویس ها را مشاهده کنید [لیست خطاها](/doc/generic-errors.html)
