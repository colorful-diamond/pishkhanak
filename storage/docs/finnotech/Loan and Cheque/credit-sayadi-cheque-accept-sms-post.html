```credit:sms-sayad-accept-cheque:post``` * **رویکرد: ** ```AUTHORIZATION_SMS``` * **<a href="ResponsecodesCredit.pdf"> دانلود مستندات ریسپانس کد ها</a> ** ```````````````` {address}/credit/v2/clients/{clientId}/users/{user}/sms/sayadAcceptCheque?trackId={trackId} ```````````````` <div class="sandboxaddress sandbox" id="sandbox" style="float:right" style="display: none" Headers مقادیر زیر باید در هدر قرار بگیرد ```````````````` Authorization : Bearer {Token} ```````````````` * برای فراخوانی این سرویس لازم است پس از دریافت اجازه ی دسترسی از کاربر، با توکن Authorization SMS سرویس را فراخوانی نمایید. #### URI Parameters * **clientId:**شناسه کلاینت * **user :** (اجباری) کد ملی کاربر که باید رشته عددی به طول ۱۰ رقم باشد #### Query Parameters * **trackId:** اختیاری (string) &#x202b; کد پیگیری، رشته ای اختیاری با طول حداکثر ۴۰ کاراکتر شامل حرف و عدد. در صورت ارسال trackId، فراخوانی سرویس خود را با همین مقدار استعلام و پیگیری کنید.(در گزارش فراخوانی سرویس ها با همین رشته نتیجه را ببینید). در صورتیکه که این فیلد را ارسال نکنید یک رشته UUID برای این فراخوانی در نظر گرفته میشود و در پاسخ فراخوانی برگردانده میشود. * example: ```getUserinfo876543``` #### Body ##### Example: ```````````````` { "sayadId": "6230008888888575", "accept": "1", "acceptDescription": "تایید", "acceptor": { "idCode": "1111111111", "shahabId": "2000014000000000", "idType": "2" }, "acceptorAgent": { "idCode": "0055555866", "shahabId": "2000014000000000", "idType": "1" } } ```````````````` * **sayadId:** (اجباری) شناسه صیاد چک , * **acceptDescription:** (اجباری) شرح تایید که این فیلد فقط میتواند شامل حروف فارسی باشد , * **accept:** (اجباری) وضعیت تایید چک که 0 به معنای رد و 1 به معنای تایید * **acceptor:** اطلاعات مشتری تائید کننده چک. این مشتری می تواند حقیقی یا حقوقی باشد و باید چک به نام او کشیده شده یا انتقال داده شده باشد, * **idCode:** (اجباری) کد شناسایی . مشتری حقیقی کدملی (اختیاری) ، مشتری حقوقی شناسه ملی شرکت * **shahabId:** (اجباری) کد شهاب , * **idType:** (اجباری) نوع کد شناسایی با ملاحضات: مشتری حقیقی 1 ،مشتری حقوقی 2 * **acceptorAgent:** در صورتیکه ثبت کننده تائید، نماینده شخص حقوقی یا وکیل شخص گیرنده چک باشد این داده اجباری خواهدبود , ####Results Format ##### Successful result format (status code 200) ```````````````` { "trackId": "7113b244-3c01-44f1-b395-e3ab464cfeaa", "result": { "message" :"تایید چک با موفقیت انجام شد", "referenceId" : "11111111111111" }, "status": "DONE" } ```````````````` * **result:** آبجکتی از پاسخ سرویس شامل: * **message:** پیغام مبنی بر تایید چک * **referenceId:** کد رهگیری * **status: ** وضعیت فراخوانی سرویس * DONE: فراخوانی موفق سرویس * FAILED: فراخوانی ناموفق سرویس * **trackId: ** کد پیگیری، اگر ارسال شده باشد همان مقدار و در غیر اینصورت یک رشته تصادفی تولید و برگردانده میشود * **error:** جزییات خطا (در صورت بروز خطا) #### UnSuccessful result format در اینجا میتوانید لیست خطاهای سرویس ها را مشاهده کنید [لیست خطاها](generic-errors.html)
